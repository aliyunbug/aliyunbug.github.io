# 🦋 BUG博物馆

> 云服务问题标本收藏馆

一个独特的云服务问题记录平台，将每一个BUG视为珍贵的标本，用收藏和研究的态度来对待它们。

![License](https://img.shields.io/badge/license-MIT-gold)
![Theme](https://img.shields.io/badge/theme-museum-8B4513)

## 🎨 设计理念

**BUG博物馆**采用独特的博物馆主题设计，将传统的BUG追踪转变为一种有趣的收藏体验：

- 🦋 **标本收藏** - 像收集昆虫标本一样记录每一个BUG
- 🏛️ **展览大厅** - 精美的卡片式展示，如同博物馆陈列
- 📚 **分类目录** - 按产品分类，系统化管理
- 🔬 **深入研究** - 详细记录BUG特征和解决方案
- ⭐ **稀有度系统** - 传说级、史诗级、稀有级、普通级

## ✨ 核心特色

### 🎭 独特的视觉风格

- **博物馆配色** - 深棕色木质感 + 金色装饰
- **羊皮纸背景** - 复古纸张质感
- **标本卡片** - 精美的边框和稀有度标识
- **浮动动画** - 顶部装饰虫子的飘动效果
- **响应式设计** - 完美适配各种设备

### 🦋 标本管理系统

- **添加标本** - 详细记录BUG信息
- **稀有度分级** - 4个等级区分问题严重程度
  - 🔴 传说级 (严重)
  - 🟠 史诗级 (高)
  - 🟡 稀有级 (中)
  - 🟢 普通级 (低)
- **分类管理** - 10+ 云服务产品分类
- **标签系统** - 自定义标签便于检索

### 🔍 强大的筛选功能

- **分类筛选** - 按产品类型过滤
- **稀有度筛选** - 按严重程度过滤
- **全文搜索** - 搜索标本名称、描述、标签
- **一键重置** - 快速清除所有筛选条件

### 💾 收藏管理

- **本地存储** - 所有数据保存在浏览器本地
- **导出收藏** - 导出为JSON文件备份
- **导入收藏** - 支持替换或合并导入
- **清空收藏** - 一键清空所有标本

## 🚀 快速开始

### 在线使用

直接在浏览器中打开 `index.html` 文件即可开始您的收藏之旅。

### 本地部署

```bash
# 克隆项目
git clone https://github.com/yourusername/bug-museum.git

# 进入目录
cd bug-museum

# 使用HTTP服务器运行
python -m http.server 8000
# 或
npx http-server
```

访问 `http://localhost:8000`

## 📖 使用指南

### 1️⃣ 添加新标本

1. 点击"添加新标本"按钮
2. 填写标本信息：
   - **分类** - 选择云服务产品
   - **稀有度** - 评估问题严重程度
   - **标本名称** - 给BUG起个名字
   - **详细描述** - 记录BUG特征
   - **研究笔记** - 记录解决方案
   - **标签** - 添加相关标签
3. 点击"收藏标本"完成添加

### 2️⃣ 浏览收藏

- 在**展览大厅**查看所有标本
- 每个标本卡片显示：
  - 产品图标和稀有度徽章
  - 标本名称和分类
  - 详细描述
  - 研究笔记（如有）
  - 标签和收藏日期

### 3️⃣ 筛选标本

使用筛选器快速定位：
- 点击筛选器展开/收起
- 选择分类和稀有度
- 输入关键词搜索
- 点击"重置"清除筛选

### 4️⃣ 查看目录

在**标本分类目录**区域：
- 查看各分类的标本数量
- 点击分类卡片快速筛选

### 5️⃣ 管理收藏

在页脚可以：
- 📤 **导出收藏** - 备份所有标本
- 📥 **导入收藏** - 恢复或合并收藏
- 🗑️ **清空收藏** - 清除所有标本
